# Ipea Demo — 产品开发计划

> **I have a Ipea** · 点亮身边的好活动  
> 课堂演示用可交互原型 · 非真实可用产品

---

## 一、项目概述

### 1.1 目标

基于 `IPea流程图.docx` 中的用户流程，制作一个**看起来是那么回事**的可点击原型，用于人因工程课程课堂展示。同学和老师可通过链接在手机上体验，演示者用 PPT 配合讲解。

### 1.2 原则

| 原则 | 说明 |
|------|------|
| 演示优先 | 重在传达产品想法，不追求真实可用 |
| 逻辑可简化 | 评分、登录、导航等均可 mock |
| 完全离线 | 静态资源，不依赖地图 API 或后端 |
| 易于分享 | GitHub Pages 部署，一个链接多端可开 |

### 1.3 参考项目

同学朋友的 [慢病用药小管家](https://nora5525.github.io/medication-manager/) 采用相同方案：

- 单页/静态 **HTML + CSS + JavaScript**
- 部署在 **GitHub Pages**（`用户名.github.io/项目名/`）
- 手机浏览器打开即 App 体验，电脑浏览器居中显示「手机框」

本项目的线上地址（部署后）：

**https://sq-chen.github.io/ipea-demo/**

## 二、技术方案

### 2.1 技术栈

| 层级 | 选择 | 理由 |
|------|------|------|
| 前端 | HTML + CSS + JavaScript | 与参考项目一致，零构建、易部署 |
| 部署 | GitHub Pages | 免费、公开链接、全班可访问 |
| 地图 | 静态广州底图 + CSS 定位标记 | 离线、稳定、课堂无网络也能演示 |
| 数据 | 本地 JSON / JS 内嵌 mock | 无后端 |

### 2.2 仓库信息

- **GitHub**：https://github.com/sq-chen/ipea-demo
- **本地路径**：项目根目录 `ipea-demo/`
- **演示设备**：Android 手机浏览器（Chrome 全屏）

### 2.3 视觉规范

**配色：暖橙点亮（方案 1）**

| 用途 | 色值 |
|------|------|
| 主色 | `#FF6B35` |
| 高分光晕 / ⭐ | `#FFB020` |
| 背景 | `#FAFAF8` |
| 文字 | `#1A1A2E` |
| 次要文字 | `#7B8294` |

**品牌命名**

- 界面标题 / Tab 栏：**Ipea**（短名，节省空间）
- 副标题 / 关于：**I have a Ipea · 点亮身边的好活动**

**活动类别（以 docx 为准）**

| 类别 | 图标 |
|------|------|
| 音乐 | 🎵 |
| 运动 | ⚽ |
| 展览 | 🖼️ |
| 美食 | 🍔 |
| 社交 | 💬 |

高分活动（≥4.5 分）在地图上带**金色光晕**或 ⭐ 标识。

## 三、功能范围（标准版）

### 3.1 必做（P0）

```
┌─────────────────────────────────────┐
│  地图 Tab    │  推荐 Tab  │  我的 Tab │
├─────────────────────────────────────┤
│  地图首页                             │
│  · 静态广州地图 + 活动标记             │
│  · 点击标记 → 底部简要卡片             │
│  · 高分活动金色光晕                    │
│  · 类别筛选（五类多选）                │
├─────────────────────────────────────┤
│  活动详情页                           │
│  · 头图 / 评分 / 时间地址 / 虚拟评论   │
│  · 【想去】【分享】                    │
│  · 【导航】(mock) 【打卡】【写评价】   │
├─────────────────────────────────────┤
│  登录弹窗（一键 mock 登录）            │
├─────────────────────────────────────┤
│  轻量分步引导（气泡提示，可跳过）       │
└─────────────────────────────────────┘
```

### 3.2 建议做（P1）

| 功能 | 说明 |
|------|------|
| 推荐 Tab | 列表视图 +「在地图上查看」跳转 |
| 我的 Tab | 想去清单 / 我的活动 / 我的评价（前端演示） |
| 筛选结果提示 | 「找到 N 个活动」+ 清除筛选 |
| Toast 反馈 | 「已加入想去清单」等轻提示 |

### 3.3 不做 / 仅 PPT 口头说明

- 真实 GPS 定位与地图 SDK
- 真实购票与支付
- 推送通知
- 积分 / 勋章后端逻辑
- md 方案中的 50 赞收录、长期/快闪/商家三类体系
- 全屏多页 onboarding

## 四、用户流程（以 docx 为主）

### 4.1 主流程（课堂演示脚本）

```
打开 App → 地图首页（广州 + 活动标记）
    │
    ├─ 筛选类别 → 地图刷新
    ├─ 推荐 Tab → 点击活动 → 详情
    └─ 拖动/点击标记 → 简要卡片 → 查看详情
              │
              ▼
         活动详情页（评分、评论、介绍）
              │
    ┌─────────┼─────────┐
    │         │         │
  返回地图   想去(需登录)  分享
              │
              ▼
         导航(mock) → 到达现场 → 打卡
              │
              ▼
         写评价 → 地图标记评分更新 / 高分点亮
```

### 4.2 轻量引导（非全屏）

不在启动时展示整页引导，改为 **5–6 步气泡提示**：

1. 看地图上的活动标记
2. 试试筛选按钮
3. 点击一个标记
4. 进入详情，点「想去」
5. 打卡与写评价
6. 看高分活动的金色光晕

每步：按钮旁一行说明 +「下一步」，可随时「跳过」。

## 七、开发阶段

按模块推进，不硬性卡时间节点；每完成一版可本地预览或推送后在线体验。

| 阶段 | 内容 | 可验收结果 |
|------|------|------------|
| **V0** | 项目骨架 + 计划文档 | 目录结构、PLAN.md |
| **V1** | 地图首页 + mock 数据 | 广州地图上可见标记，点击弹出卡片 |

## 八、部署指南（GitHub Pages）

### 8.1 首次推送

```bash
cd ipea-demo
git init
git add .
git commit -m "init: Ipea demo project plan and skeleton"
git branch -M main
git remote add origin https://github.com/sq-chen/ipea-demo.git
git push -u origin main
```

### 8.2 开启 Pages

1. 打开 https://github.com/sq-chen/ipea-demo/settings/pages
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main`，文件夹选 `/ (root)`
4. 保存后等待 1–2 分钟，访问 https://sq-chen.github.io/ipea-demo/

### 8.3 后续更新

```bash
git add .
git commit -m "描述本次改动"
git push
```

## 九、课堂演示建议

### 9.1 PPT 可包含

- 产品痛点（来自 Obsidian 方案 md）
